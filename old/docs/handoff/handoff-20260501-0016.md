- 重要な判断:
  - Companion Merge 画面はソフトウェアエンジニア向けの merge gate として扱う。
  - `target-branch-drift` がある場合は `Merge Selected Files` を無効化し、先に `Sync Target` を実行する設計にした。
  - `Sync Target` は target branch が fast-forward している場合だけ許可し、target と Companion の同一ファイル競合は一時 worktree で事前検出する。
  - `baseSnapshotCommit` は stale window から巻き戻らないように、汎用 `updateSession` では更新せず、Sync Target 専用の `updateSessionBaseSnapshot` で更新する。
  - Sync Target 後は選択状態を保持し、選択済みファイルが消えた場合でも未選択ファイルを自動で全選択しない。
  - Audit Log の重い detail 読み込みは DB 格納設計から見直す必要があるため、このスコープでは実装せず後続タスクへ切り出す。
