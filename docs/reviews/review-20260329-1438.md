# UI/UX レビュー — WithMate

- **レビュー日時**: 2026-03-29 14:38 (JST)
- **対象**: アプリケーション全体のUI/UX
- **観点**: アクセシビリティ、レスポンシブデザイン、ユーザーフィードバック、一貫性、インタラクション、エラーハンドリング、パフォーマンスUX、情報設計、フォーム/入力、Electronウィンドウ管理

---

## 指摘一覧（11件）

### 🔴 重大 (Critical / High)

#### 1. モーダルのキーボード操作

**問題**: 全モーダルで ESC キーによる閉じ操作、フォーカストラップ、開いた際の自動フォーカスが欠如している。

**影響**: キーボードのみで操作するユーザーがモーダルを閉じられない、またはモーダル背後のコンテンツに誤ってフォーカスが当たる。

**改善案**:
- モーダル表示時に内部の最初のフォーカス可能要素へ自動フォーカス
- `keydown` で `Escape` キーを検知してモーダルを閉じる
- `Tab` / `Shift+Tab` がモーダル外に出ないようフォーカストラップを実装（`focus-trap` ライブラリ等）

---

#### 2. ボタングループのキーボードナビゲーション

**問題**: Listbox やグループ系コントロールで正しい ARIA パターンが使われておらず、矢印キーによる選択操作が未対応。

**影響**: スクリーンリーダーユーザーや、マウスなしで操作するユーザーがグループ内の選択肢を切り替えられない。

**改善案**:
- `role="listbox"` + `role="option"` の組み合わせを正しく適用
- `ArrowUp` / `ArrowDown` で選択肢間を移動し `aria-selected` を更新
- WAI-ARIA Authoring Practices の Listbox パターンを参照

---

### 🟡 中程度 (Medium)

#### 3. ARIA live region の過剰使用

**問題**: 4つの `aria-live` リージョンが同時に存在し、複数が同時に更新される可能性がある。

**影響**: スクリーンリーダーが複数のアナウンスを重複して読み上げ、ユーザーが情報を追いきれなくなる。

**改善案**:
- ライブリージョンを1〜2個に集約し、優先度に応じて `polite` / `assertive` を使い分ける
- 同時更新が発生しないよう、更新タイミングをキューで管理する

---

#### 4. Composer の Ctrl+Enter 送信ブロック時の視覚フィードバック欠如

**問題**: 送信が無効な状態（空メッセージ等）で `Ctrl+Enter` を押しても、なぜ送信できないのかの視覚フィードバックがない。

**影響**: ユーザーが何が問題なのかを把握できず、混乱する。

**改善案**:
- 送信不可の理由をインライン表示（例:「メッセージを入力してください」）
- 送信ボタンに `title` 属性でブロック理由を付与
- 短時間のシェイクアニメーション等で「操作を受け付けていない」ことを伝える

---

#### 5. DiffViewer のキーボードスクロール不可

**問題**: DiffViewer のスクロールペインにキーボードフォーカスが当たらず、矢印キー・Page Up/Down でのスクロールができない。

**影響**: マウスなしで Diff 内容を読むことができない。

**改善案**:
- スクロール可能なコンテナに `tabindex="0"` を付与
- フォーカス時に視覚インジケーターを表示

---

#### 6. カラーコントラスト計算が WCAG 非準拠

**問題**: コントラスト判定に WCAG 準拠のコントラスト比（4.5:1 / 3:1）ではなく輝度閾値を使用している。

**影響**: 実際には WCAG AA 基準を下回るコントラストが「合格」と判定される可能性がある。

**改善案**:
- WCAG 2.1 の相対輝度計算式と 4.5:1（通常テキスト）/ 3:1（大テキスト）の比率判定を実装
- または `color-contrast` 等のライブラリを使用

---

#### 7. 1400px 以下でのレスポンシブ対応

**問題**: ウィンドウ幅 1400px 付近で右ペインが代替手段なく消失する。

**影響**: ユーザーが右ペインのコンテンツにアクセスする方法がなくなる。

**改善案**:
- 右ペインを折りたためる/展開できるトグル UI を追加
- または小さいウィンドウでは単一カラムレイアウトへ切り替え、タブ等で切り替え可能にする

---

#### 8. ウィンドウ最小サイズが大きすぎる

**問題**: メインウィンドウの最小幅 1040px、DiffViewer の最小幅 1180px は、小画面ディスプレイや分割表示環境では使用できない。

**影響**: 一部のユーザーはアプリを実用的に使用できない。

**改善案**:
- メインウィンドウの最小幅を 800px 程度まで縮小し、レスポンシブレイアウトで対応
- DiffViewer は横スクロールを許容するか、最小幅を下げる

---

### 🔵 軽微 (Low)

#### 9. Composer 添付ファイルのオーバーフロー

**問題**: 多数のファイルを添付した場合に高さ制限がなく、スクロールも提供されていない。

**影響**: 多数添付時にUIレイアウトが崩れ、送信ボタンなどが画面外に押し出される可能性がある。

**改善案**:
- 添付ファイルリストに `max-height` と `overflow-y: auto` を設定
- または表示件数を制限し「他N件」のように省略表示する

---

#### 10. エラーバウンダリの復帰手段がない

**問題**: エラーバウンダリのフォールバック UI にリトライ・リロードボタンが存在しない。

**影響**: エラー発生時にユーザーがアプリを回復させる手段がなく、手動でウィンドウを閉じて再起動するしかない。

**改善案**:
- フォールバック UI に「再試行」ボタンを追加し、`componentDidCatch` / `reset` 機能と連携させる
- 致命的エラーの場合は「アプリを再起動」ボタンを Electron の `app.relaunch()` と連携して表示する

---

#### 11. PathMatch の Tab キー挙動の競合

**問題**: PathMatch コンポーネントの Tab キーハンドリングが標準のオートコンプリート UX パターンと競合している。

**影響**: ユーザーが Tab で候補を選択しようとしたとき、または次のフィールドに移動しようとしたときに意図しない動作が起きる。

**改善案**:
- Tab は「次のフィールドへ移動」の標準動作を維持する
- 候補選択には `Enter` または `ArrowDown` + `Enter` を使用する（WAI-ARIA Combobox パターン参照）

## ✅ 良い点

- ARIA 属性が全体的に適切に使われている
- `visually-hidden` ユーティリティが正しく実装されている
- `role="alert"` によるエラー通知が適切に機能している
- 無効状態（disabled）のスタイリングが視覚的に明確
