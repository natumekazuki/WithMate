- 重要な判断:
  - Sessionの実行状態はRunから導出し、Session rowへprojection保存しない。Home recent orderだけは`last_activity_at` projectionを持ち、keyset paginationと1回のset queryでRun導出値を返す。
  - Session / Message / Runと補助11 tableの合計14 tableを初期永続化schemaとする。
  - RunOutput payloadはSQLite BLOBだけへ保存する。上限は1件16 MiB、Run 64 MiB、Session 256 MiB、app全体1 GiBで、disk reserveもcommit前に検証する。
  - payload準備はwrite transaction外で行い、terminal outcome / final Messageをdetail BLOBより先にcommitする。payload読込はpreview / chunk / exportへ分離する。
  - streaming assistant deltaと未解決interactionはメモリだけに保持する。draftはper-Run 4 MiB / app全体32 MiB、chunk append、coalescing、Renderer backpressureを契約化した。
  - userによるSession削除は、non-terminal Runがない場合に限りchild Session subtreeを物理削除する。初期版はlocal-onlyでProvider側削除を保証しない。WAL / incremental vacuum / secure deleteのmaintenance境界を定義した。
  - 旧DBからdata migration、import、compatibility readerを実装しない。旧DB fileは参照・変更・自動削除しない。
  - completed persistent Codex ThreadはApp Server再起動後にread / resumeできる。
  - active Turn中のstdio App Server異常終了では同じTurnが`interrupted`となり、未確定assistant draftは復元しない。
  - 外部会話作成は`creating` ProviderBinding intentを先にcommitし、受理不明な`thread/start`を自動再送しない。
  - `waitChild` / `waitAny` / `waitAll`はready handleだけを返すread-only操作とし、ChildResultの親tool result化、初回回収、冪等応答は単一Deliveryの`collectChildResult`だけが記録する。
  - child startの冪等keyはparent Sessionへscopeし、child delete後の再送で別childを作らない。
  - child concurrencyはroot上限に加え、通常Run / Auxiliaryも含むapp全体とProvider runtime別hard capを同じadmissionで検証する。
  - closed Delegationは明示retryだけ原子的にactiveへ戻せる。ProviderBinding作成失敗時はDispatchも`aborted`へ収束させる。
  - completed IdempotencyRecordは期限後にresponse参照を消去した`expired` tombstoneとなり、scope Session削除までkeyを再利用させない。
  - redaction判定不能なRunOutput payloadは`omitted_redaction`としてBLOBへ保存しない。
